# feature-init 交互拆分 — 设计文档

**日期:** 2026-07-07
**状态:** approved

## 1. 目标

将 feature-init 步骤1的「一次性问两个问题」拆为两轮独立交互，每轮最多一个提问。遵循「一次一问」原则。

## 2. 当前行为

步骤1 一次性展示两个问题：
1. 需求描述（1-3 句话）
2. 任务 ID（可选，不指定则自动生成）

## 3. 目标行为

### 步骤1：收集需求描述

自然语言向用户提问：
> 请用 1-3 句话简要描述你的需求（需要开发的功能或需要调整的存量功能）。

用户回复后，将描述保存到 `inputs/requirement.md`。

### 步骤2：收集任务 ID（可选）

自然语言向用户提问：
> 是否需要指定任务 ID？不指定则自动生成为 `FEAT-YYYYMMDD-NNN` 格式。

- 用户指定 → 使用该 ID
- 用户回复「不用」/「自动」/「不需要」等 → 自动生成

### 后续步骤（原步骤 2~5 依次后移）

- 步骤3：创建任务工作区（原步骤2）
- 步骤4：创建初始文件（原步骤3）
- 步骤5：确认展示（原步骤4）
- 步骤6：提示下一步（原步骤5）

## 4. 改动清单

| 文件 | 改动 |
|------|------|
| `skills/feature-init/SKILL.md` | 步骤1 拆为两轮交互（步骤1+步骤2），原步骤2~5 后移一位 |

## 5. 不变内容

- 自然语言交互（feature-init 为开放式输入场景，不使用 AskUserQuestion 选项卡）
- 集成契约中的入口、入参、输出、完成标准
- 脚本调用逻辑
