# 相关设计追溯评审

## 适用条件

所有设计类型必选。只评审 Draft 声明采用的需求、Artifact、代码事实和约束；不存在相关设计输入时，检查其独立边界是否明确。

## 评审规则

- 每个关键设计结论都能追溯到目标、事实、约束或明确取舍。
- Draft 不得静默弱化、扩大或矛盾其声明采用的输入。
- 追溯关系以可理解为准，不强制表格、ID 密度或固定上游类型。
- 缺少必要输入时指出具体缺口及其对当前设计的影响。

## 检查项

- [ ] 目标和范围均有设计落点。
- [ ] 关键规则、接口、状态、风险或约束没有被遗漏或改义。
- [ ] 新增假设和取舍在 Draft 中显式表达。
- [ ] 相关 Artifact 的版本或语义没有被错误引用。
- [ ] 没有相关设计输入时，独立使用边界和限制已经说明。
