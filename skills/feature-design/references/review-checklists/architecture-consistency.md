# 架构一致性评审

## 适用条件

方案设计必选。

## 评审规则

- 架构只承担当前目标、约束和质量属性需要的复杂度。
- 边界、模块职责、依赖方向、接口和数据所有权相互一致。
- 评审矛盾、缺失的承重决策和实质风险，不追求架构形式完整。

## 检查项

- [ ] 系统与外部边界的责任没有重叠或空洞。
- [ ] 每个模块具有清晰职责和稳定接口。
- [ ] 依赖方向与数据所有权不存在循环或冲突。
- [ ] 关键调用的成功、失败和恢复路径闭合。
- [ ] 外部集成的超时、重试、幂等和降级语义一致。
- [ ] 质量属性场景与架构选择对应。
- [ ] 推荐方案的关键代价和残余风险已表达。
