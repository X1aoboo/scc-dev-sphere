# 兼容性测试评审

## 适用条件

客户端、Schema、协议、版本、平台或滚动部署形成兼容风险时执行。

## 评审规则

- 兼容范围和不支持范围都有明确版本或组合边界。
- 覆盖生产中真实存在的混合版本、升级和降级窗口。
- 默认值、未知字段和能力协商的行为可验证。

## 检查项

- [ ] 支持的客户端、服务、Schema、协议和平台组合明确。
- [ ] 新旧版本双向交互和混合部署场景覆盖。
- [ ] 新增、删除、重命名字段及默认值行为覆盖。
- [ ] 升级、降级、回滚和数据残留场景覆盖。
- [ ] 能力协商、未知值和不支持版本失败语义明确。
- [ ] 兼容矩阵与回归范围相称且可执行。
