# 安全测试评审

## 适用条件

当前测试目标涉及身份、权限、信任边界、外部输入、秘密、敏感数据或滥用风险时执行。

## 评审规则

- 安全测试对应具体威胁和控制措施。
- 覆盖控制失败、绕过和负向路径，而不只验证正常认证。
- 测试方法、数据和环境不会引入秘密或破坏性风险。

## 检查项

- [ ] 身份认证失败、过期和冒用场景已覆盖。
- [ ] 横向、纵向越权和对象级授权已验证。
- [ ] 输入验证、注入、解析和边界值有负向测试。
- [ ] 秘密和敏感数据不会在日志、错误或响应中泄漏。
- [ ] 重放、枚举、滥用和速率限制场景适用时覆盖。
- [ ] 安全审计事件和告警结果可验证。
