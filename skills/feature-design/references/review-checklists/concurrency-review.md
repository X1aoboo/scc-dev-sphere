# 并发一致性评审

## 适用条件

并发写入、异步执行、顺序、共享状态、重复交付或重试可能改变正确性时执行。

## 评审规则

- 用具体竞争时序验证正确性，不只罗列锁或事务技术。
- 明确原子性、隔离、顺序和幂等保证作用于哪些状态。
- 冲突、重复和中断后的恢复结果必须可观察。

## 检查项

- [ ] 共享可变状态和竞争参与者已经识别。
- [ ] 读改写、检查后执行等竞态有明确处理。
- [ ] 原子性和事务边界覆盖全部相关写入。
- [ ] 顺序、重复、乱序和至少一次交付语义明确。
- [ ] 幂等键、唯一约束或去重窗口足以成立。
- [ ] 冲突、取消、超时和进程中断后的恢复一致。
