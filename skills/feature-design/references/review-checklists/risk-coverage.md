# 风险覆盖评审

## 适用条件

测试设计必选。

## 评审规则

- 根据 Draft 声明采用的目标、相关设计和风险检查覆盖，不要求固定上游 Artifact 组合。
- 每个关键风险都有相称测试层级、可观察结果和失败判定。
- 不可测项必须显式处理，不能默认视为接受。

## 检查项

- [ ] 最高影响和最高不确定性风险均有测试策略。
- [ ] 关键业务结果、接口、状态和规则有可观察验证。
- [ ] 负向、边界、故障、恢复和回归场景与风险相称。
- [ ] 测试层级选择避免重复和不必要的 E2E 依赖。
- [ ] 迁移、兼容、并发或外部集成风险按适用性覆盖。
- [ ] 不可测项包含原因、影响、缓解和责任。
- [ ] 转测准入能够阻止关键风险未验证的交付。
