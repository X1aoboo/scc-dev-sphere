# 可运维性评审

## 适用条件

生产运行、诊断、发布、回滚、容量或值守责任受到影响时执行。

## 评审规则

- 运维信号必须支持具体判断和操作，而不是泛泛要求“加日志”。
- 发布、控制、诊断和恢复覆盖关键失败模式。
- 运行责任和人工动作具有清晰入口与安全边界。

## 检查项

- [ ] 关键状态、错误和业务结果有可查询日志或指标。
- [ ] 告警具有阈值、含义和可执行响应。
- [ ] 功能开关、灰度、暂停和恢复策略明确。
- [ ] 发布、回滚和故障诊断步骤可操作。
- [ ] 容量、积压、延迟和资源耗尽信号充分。
- [ ] owner、权限和 runbook 影响已经说明。
