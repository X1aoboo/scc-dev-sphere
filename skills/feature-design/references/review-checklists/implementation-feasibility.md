# 实现可行性评审

## 适用条件

实现设计必选。

## 评审规则

- 每项代码影响建立在真实仓库事实和存在的接缝上。
- 关键行为已经设计到足以实现，不把核心决定留给编码临场处理。
- 实现细节与声明采用的相关设计、接口和约束一致。

## 检查项

- [ ] 受影响模块、文件和关键符号可定位。
- [ ] 接口、类型、输入输出和调用顺序足够明确。
- [ ] 错误、取消、超时和重试行为可实现。
- [ ] 事务、并发、顺序和幂等边界一致。
- [ ] 数据、协议、配置和部署变化具有迁移与回滚路径。
- [ ] 可观测性和公开测试接缝能够验证关键行为。
- [ ] 没有依赖未证实仓库能力的关键步骤。
