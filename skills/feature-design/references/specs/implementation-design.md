# Implementation Design

本 Spec 是最终 Implementation Design Draft 的内容合同。核心章节采用一个 Feature 级公共上下文，加上按 Solution Design 已确定微服务重复的完整详细设计单元。新增与存量微服务使用同一结构；条件章节已经纳入各微服务单元，并按实际影响展开。

Draft 表达已确认的代码级目标实现，不承担设计讨论顺序、跨服务架构设计、代码 Diff、Review 检查、开发任务编排或单元测试代码。第一版保持单一 Feature 级文档。

语义图不是必选产物，不要求 Draft 必须包含图示，也不规定图示数量。组件或类关系、调用时序、控制流、数据流或状态转换适合使用图示时，必须采用 Mermaid；禁止使用 ASCII 字符模拟语义图。界面设计、视觉稿等不适合 Mermaid 的内容不强制绘图。Markdown 表格、目录树和代码片段不属于 ASCII 语义图。

说明文档目的、当前 Feature、目标读者、实现设计覆盖范围、实际采用的输入和关键实现约束。业务背景和价值只保留理解实现边界所需的摘要。
